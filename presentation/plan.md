UniTrack = self-hosted server for tracking test execution + coverage over time (Allure-meets-
Codecov, now polyglot). Web dashboard + CLI uploader/gate. This refresh folds in everything shipped
this session. Reuses the existing Selenium `ScreenshotTourTest` + demo dataset + dark theme.

## Aspects  (★ = new or materially changed since run-1)

1. **hero / board** — all-projects health (`/`): pass%, coverage, flaky, gate per repo, **failing
   gates first**, with the **"broken since · 2w · N runs" rot signal** on regressed projects — *Selenium*
2. **project-overview ★** — one project (`/projects/{id}`): gate donut + the **time-aware trend chart**
   (true time x-axis, cadence = dot density, regression onset annotated) + recent runs — *Selenium*
3. **coverage** — per-file / per-package line coverage for the latest run — *Selenium*
4. **flaky** — flaky detection: passed *and* failed on the same commit, rates, quarantine — *Selenium*
5. **clusters** — failures grouped by normalized signature + recurring failures — *Selenium*
6. **AI root-cause ★ (NEW — headline)** — the click-to-run **"Analyze with AI"** card on a cluster:
   LLM root cause + concrete fix + confidence (Spring AI / Anthropic, BYOK; live on the deploy) — *Selenium*
7. **load tests ★ (NEW)** — the **Load tests** tab (`/projects/{id}/perf`): latency p50/p90/p95/p99,
   throughput, pass/fail, regression-vs-baseline — multi-format perf ingest (JMeter JTL, JMH, k6,
   Gatling, …) — *Selenium*
8. **test timing** — unit-test performance: slowest tests + suite-time trend (`/performance`) — *Selenium*
9. **run-detail** — a single run (`/runs/{id}`): failures with stacktraces + quality-gate verdict — *Selenium*
10. **quality-gate (CLI)** — the CI payoff: `unitrack-cli gate …` machine-readable PASS/FAIL
    exit-code contract — *Freeze (or HTML terminal still, as in run-1)*
11. **outcome / owners ★** — cross-project **owner accountability board** (`/owners`): who owns the
    rot, per-owner failing/flaky rollup (#287) — *Selenium*
